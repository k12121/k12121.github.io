  • 简略复习 
 

  • ioc

    • ioc（容器）是控制反转，也就是说在之前创建java对象的时候，由程序员自己创建，而在spring中，对象的创建由spring去管理，也就是“控制”已经由spring进行了，创建对象的权力被“反转”了。（这种控制反转的好处是，降低了代码之间的耦合度，提高了代码的可维护性和可测试性。它还提供了依赖注入的机制，使得对象之间的依赖关系更加灵活和可配置。）

  • DI

    • DI指的是依赖注入，假设A类里面使用了B类里面的方法，可以认为A类依赖B类，在实际开发中，也可以认为controller依赖service,service依赖Dao,那么注入的就是IOC里面的对象或者说具体值（依赖注入的好处是，它降低了类之间的耦合度，使得类的设计更加灵活和可维护。它还使得单元测试更容易，因为可以更方便地替换依赖对象来进行测试。）但是在多例情况下会失败，因为不存在三级缓存，在使用构造器注入的时候，也会失败，因为在这个时候，实例还没有实例化成功，也不存在三级缓存。

  • 依赖冲突

    • 假设由两个类分别为A类和B类，在A类里面注入了B类的实例，在B类里面也注入了B类的实例，这样的情况叫做循环依赖，这种情况可以实现，通过spring的三级缓存实现

  • Spring三级缓存

    在 Spring 框架中，有一个叫做”三级缓存”的概念，它与 Spring Bean 的生命周期相关。下面是关于三级缓存的解释：

    1. Singleton Objects Cache（单例对象缓存）：这是 Spring 中的第一级缓存。当 Spring 容器创建一个单例 Bean时（单例 Bean 是一种 Bean 的作用域，表示在整个应用程序中只会创建一个实例。） ，会将其放入此缓存中。如果在创建过程中发现循环依赖，Spring 会在缓存中找到该 Bean 的代理对象，以解决循环依赖的问题。

    2. Early Reference Objects Cache（早期引用对象缓存）：这是 Spring 中的第二级缓存。当 Spring 创建一个单例 Bean 的实例时，会将其放入此缓存中。这样，在创建过程中如果有其他的 Bean 需要引用该对象，可以提前使用缓存中的实例，而不是等待该对象完全创建完成。

    3. Singleton Factory Map（单例工厂映射）：这是 Spring 中的第三级缓存。当 Spring 创建一个单例 Bean 的实例时，会将其放入此缓存中。这个缓存中存储的是 Bean 的工厂对象，可以用于解决循环依赖的问题。

    这三级缓存在 Spring 的 Bean 实例化过程中起到重要的作用，它们通过缓存对象的引用，解决了循环依赖的问题，并提供了更高效的对象创建和引用的方式。但需要注意的是，这些缓存只在单例作用域的 Bean 创建过程中使用，对于其他作用域的 Bean，不会使用这些缓存。
